README 의 거짓 예제를 고치고, 문서 주장을 테스트로 못 박습니다.

v0.0.3 부터 README 는 이렇게 적고 있었습니다.

```ts
detectBest("3333-12-3456789");
// → "카카오뱅크" "new" "high"   ← 거짓
```

실제 반환값은 **신한은행 / low** 입니다. 카카오뱅크 13자리 패턴(`3333-XX-XXXXXXX`)은 PDF 에 식별 코드가 없어 길이 적합(3점)만으로 경쟁하고, 같은 3점에서 `priority` 가 높은 신한은행이 1순위가 됩니다.

- 두 README 의 빠른 시작 예제를 실제로 `high` 가 나오는 `1002-123-456789` (우리은행) 으로 바꿨습니다.
- 식별 코드가 없는 기관의 동작을 "한계" 로 명시하고, `include` 로 좁히는 방법을 안내합니다.
- **`src/readmeExamples.spec.ts` 신설** — README 의 모든 코드 예제가 주장하는 값을 단언합니다. 문서가 다시 코드와 어긋나면 CI 가 실패합니다.

런타임 코드 변경은 없습니다 (문서·테스트만). `detect()` 출력은 그대로입니다.
