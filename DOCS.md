## 0. 식별 위치 패턴 (개발자 핵심)

CMS 계좌번호는 기관·자릿수에 따라 **계정과목 코드(과목코드)의 위치가 다릅니다.** 다음 4가지 패턴 중 하나로 떨어집니다.

| 패턴 | 구조 | 과목코드 위치 | 대표 예 |
|---|---|---|---|
| **A. 앞 (Front)** | `과목 - 일련 - 검증` | 1~3번째 자리 | 차세대 신계좌 대부분 (산업 14, 하나 12 차세대, 농협 13, 부산 13, 경남 13, 신협 12, 새마을금고 13, 우체국 12, 신한 12, 토스 12 등) |
| **B. 가운데 (Middle)** | `점번호 - 과목 - 일련 - 검증` | 4~5번째(또는 5~6, 7~8) | 구계좌 다수 (산업 11, 기업 12, 국민 12, 하나 11, 우리 구상업/한일/평화, 한국씨티 11/13, 부산 12, 광주 12, 제주 10, 전북 12, 경남 12, 부국 11, 케이프 11 등 / 농협 14 7~8번째) |
| **C. 뒤 (End)** | `일련 - 검증 - 과목` | 마지막 2자리 | 키움 10, 아이엠증권 10, 메리츠 10, 하나금융투자 10, 하나증권 14, HSBC 12(끝 3자리) |
| **D. 없음** | 과목코드 미사용 | 없음 | 삼성/한국투자/NH/SK/대신/한화/유진/카카오페이/신영 등 다수 증권사, 신협 10/11, K뱅크 13, 카뱅 등 |

> **반드시 [기관 코드 + 자릿수] 조합으로 매칭하세요.** 같은 기관도 자릿수마다 체계가 완전히 다릅니다.

## 4. 개발자 체크리스트

1. **매칭 키**: `대표코드 + 자릿수 + 첫 1~3자리 패턴` (특히 인터넷전문은행/가상계좌).
2. **과목코드 위치**: 같은 기관도 자릿수마다 위치가 다름. 단일 정규식만으로는 충분하지 않음.
3. **수협 분기 처리**:
   - 11자리: 4·5번째 자리가 43~45,47,49,59,61~64,66~68,74,75,78,81~85,93 → 030, 그 외 → 007
   - 12자리: 1번째 자리가 2/7/9 → 030, 그 외 → 007
   - 14자리: 1·2·3번째 자리가 493, 481~489 → 030, 그 외 → 007
   - 수협중앙회 12자리 (반대 방향): 1번째 자리가 2/7/9면 030 유지, 그 외 → 007
4. **농협 차세대 13자리**: 과목코드 (앞 3자리) 만으로 보통/저축 등을 분류. 끝자리(계좌구분) 의 1·2=구가상, 3·4·5=입금전용(적금·신탁) 규칙은 *과목코드가 적금/신탁* (304·310·314·321·324·334·345·347·349·359·380 / 028·031·043·046·079·081·086·087·088 / 354·360·384·394·398) 일 때만 적용. 일반 과목코드(301·302·312·306·305·317 / 351·352·356·355)는 끝자리 무관.
5. **K뱅크 식별**: 10자리 첫자리=9(입금전용), 14자리 첫자리=7 또는 9(간편송금/안심계좌).

## 6. KFTC 표준은행코드 vs CMS 참가기관 코드

두 코드 체계는 모두 KFTC 가 운영하지만 **별도 namespace** 다. 같은 institution 이 두 체계에서 다른 코드를 가질 수 있다.

| 항목 | CMS 참가기관 코드 (라이브러리 `Institution.code`) | 표준은행코드 (라이브러리 `Institution.commonCode`) |
|---|---|---|
| 출처 | 금융결제원 CMS EDI PDF (이 문서) | 금융공동망 공동코드 |
| 용도 | CMS 자동이체·공과금 시스템 식별 | 일반 송금/이체 시스템 식별 |
| 라이브러리 노출 | `Institution.code` (모든 institution 에 필수) | `Institution.commonCode` (CMS 와 다를 때만 지정, optional) |

**구조적 갈래의 원인**: 2015년 외환·하나 합병 시 CMS 측은 외환의 `005` 를 통합 대표코드로 승계 → CMS 코드표 라벨 *"하나은행(구.외환,하나)"* 으로 표기. 반면 금융공동망 측은 하나은행의 `081` 을 대표코드로 유지. 라이브러리 PDF-strict 정책상 CMS 표 그대로 `code: "005"`, 표준은 `commonCode: "081"` 로 별도 노출.

### 6.2 사용 패턴

```ts
import { detectBest, institutionById } from "korean-account";

// 표준은행코드 기반 서버를 쓰는 컨슈머
const top = detectBest("161-910278-72907");
const bankCode = top?.institution.commonCode ?? top?.institution.code;
//                                              ↑ fallback (대부분 동일)
// → "081" (하나은행 표준)

// CMS namespace 컨슈머는 그대로 .code 사용
const cmsCode = top?.institution.code;
// → "005" (하나은행 CMS)
```

### 6.3 잠재 추가 mismatch — 후속 점검 대상

- **농협중앙(012)** — 표준 매핑 별도 확인 (현재 라이브러리 `commonCode` 미지정)
- **수협 007 ↔ 수협중앙 030 (2025.11 분리)** — 표준의 분리 시점·대표코드 정책 확인
- **외국계 미참가 (054 HSBC / 055 도이치 / 057 JPMC / 061 BNP파리바)** — 표준 등록 여부 확인. 활성화 시 `commonCode` 패턴으로 흡수.
- **하나은행 CMS 별칭 `081`** — PDF 표 § 3 의 "005 하나은행: 025, 033, 080, 081, 082" 별칭 set 에 `081` 포함되나, 라이브러리는 `hanaSecuritiesCma (code: "081")` 와 코드 충돌 회피 위해 `hana.aliasCodes` 에서 081 누락. 의도된 trade-off — `institutionByCode("081")` 은 *하나증권 CMA* 만 반환.
